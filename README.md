# Практична робота з Git та GitHub

Цей репозиторій створено в рамках навчального курсу для ознайомлення з системою керування версіями Git та платформою GitHub. Студенти виконують серію завдань, які послідовно охоплюють ключові концепції: від базових операцій до колаборативної розробки та автоматизації.

## Мета роботи
- Оволодіти базовими та просунутими можливостями Git і GitHub.
- Навчитися працювати з репозиторіями, гілками, конфліктами, Pull Requests, Issues, Projects та GitHub Actions.
- Здобути навички, необхідні для ефективної роботи в команді над програмними проектами.

## Опис завдань

### Завдання 1: Створення облікового запису та першого репозиторію
- Зареєструватися на GitHub.
- Створити публічний репозиторій `my-first-project`.
- Додати `README.md`, `.gitignore` (відповідно до обраної мови) та ліцензію (наприклад, MIT).

### Завдання 2: Клонування та локальна робота з репозиторієм
- Встановити Git, налаштувати ім'я користувача та email.
- Клонувати репозиторій на локальний комп'ютер.
- Додати файл `index.html` з простою HTML-сторінкою.
- Виконати коміт та відправити зміни на GitHub.

### Завдання 3: Робота з гілками (branches)
- Створити гілку `feature/add-styles`, переключитися на неї.
- Додати `styles.css` та підключити його до `index.html`.
- Виконати коміт, відправити гілку на GitHub.
- Створити Pull Request та об'єднати з основною гілкою.

### Завдання 4: Вирішення конфліктів
- Створити дві гілки: `feature/update-title` та `feature/add-footer`.
- У кожній змінити один і той самий рядок (або додати зміни, що конфліктують).
- Об'єднати першу гілку з `main`, потім спробувати об'єднати другу.
- Вирішити конфлікт вручну, зафіксувати результат.

### Завдання 5: Колаборація та Code Review
- Об'єднатися в пари, додати партнера як співавтора.
- Клонувати репозиторій партнера, створити гілку та внести зміни.
- Створити Pull Request до репозиторію партнера.
- Провести Code Review, залишити коментарі, схвалити або запросити зміни.

### Завдання 6: Використання Issues та Projects
- Створити 5–7 Issues у своєму репозиторії.
- Організувати Project Board (To Do, In Progress, Done) та розподілити Issues.
- Для кожного Issue створити окрему гілку, виконати зміни, використати ключове слово (наприклад, `Fixes #42`) у коміті.
- Стежити за автоматичним переміщенням карток на дошці.

### Завдання 7: Робота з віддаленими репозиторіями та фіксація історії
- Зробити форк цікавого відкритого репозиторію.
- Клонувати свій форк локально, додати оригінальний репозиторій як upstream.
- Створити гілку, внести змістовний внесок, відправити у свій форк.
- Створити Pull Request до оригінального репозиторію.

### Завдання 8: Використання GitHub Actions для автоматизації
- Створити директорію `.github/workflows` та файл `ci.yml`.
- Написати workflow, який запускається при кожному push і перевіряє синтаксис/тести.
- Запушити зміни та переконатися в успішній роботі.

### Завдання 9: Документування проекту
- Розширити `README.md`: додати опис, інструкції з встановлення, приклади використання, інформацію про внесок.
- Створити `CONTRIBUTING.md` з рекомендаціями для учасників.
- Додати бейджі зі станом CI до `README.md`.
- Налаштувати GitHub Pages для зручної презентації документації.

### Завдання 10: Інтеграційний проект
- Створити окремий репозиторій для фінального проекту.
- Застосувати всі вивчені практики: гілки, Issues, Project Board, Code Review, GitHub Actions.
- Підготувати презентацію, що демонструє робочий процес команди на GitHub.

## Структура репозиторію
- `README.md` – загальний опис (цей файл).
- `.gitignore` – список ігнорованих файлів.
- `LICENSE` – ліцензія MIT (або інша обрана).
- `index.html`, `styles.css` – приклади, які створюються під час виконання завдань.
- `.github/workflows/` – директорія з конфігураціями GitHub Actions.

## Як використовувати
1. Виконайте **Завдання 1**, створивши власний репозиторій (або працюйте з цим, якщо викладач надав доступ).
2. Послідовно виконуйте завдання 2–10, дотримуючись наведених кроків.
3. Кожне завдання базується на попередніх, тому важливо виконувати їх у порядку.
